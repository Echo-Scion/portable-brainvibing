- **Disambiguation (UI-Finish vs Frontend-Experience)**: Use `ui-finish` STRICTLY for visual engineering, creative polish, and aesthetics. Do NOT use this skill for analytical component tree debugging, state management, or structural layout fixing. For structural fixes, delegate to the `frontend-experience` skill.
- **Inclusive Visuals**: NEVER accept "Clone Faces." Mandate distinct facial structures and authentic attire.
- **Performance**: NEVER overuse heavy graphical filters on large scroll lists; consolidate visual effects.
- **Accessibility & Semantics**:
  - NEVER ignore "Reduce Motion" settings.
  - ALWAYS add semantic labels to icon-only buttons or custom interactive widgets (e.g., `aria-label`).
  - Ensure focus states are clearly visible for keyboard navigation.
  - Exclude decorative images from semantics or provide semantic labels for informative images (alt text equivalent).

## 🛠️ Micro-UX Standards

**Good UX Code Example:**
```javascript
// ✅ GOOD: Accessible icon button with Semantics & Tooltip
<Tooltip message="Delete project">
  <button
    aria-label="Delete project"
    disabled={isDeleting}
    onClick={handleDelete}
  >
    {isDeleting ? <Spinner /> : <TrashIcon />}
  </button>
</Tooltip>
```

**Bad UX Code Example:**
```javascript
// ❌ BAD: No Semantic label, no disabled state, no loading feedback
<div onClick={handleDelete}>
  <TrashIcon />
</div>
```
